## Naming Conventions

**Files:**
- snake_case for Python files: `main.py`
- UPPERCASE.md for important project files: `README.md`
- Lowercase for config files: `requirements.txt`

**Directories:**
- lowercase with hyphens: `.planning/`
- Standard Python: `.venv/`, `__pycache__/`

**Special Patterns:**
- `.` prefix for hidden/config directories
- No `__init__.py` (single module, not a package)

## Where to Add New Code

**New Feature:**
- Primary code: `main.py` (add functions before GUI code)
- Tests: Create `tests/` directory and `test_main.py`
- Config if needed: `pyproject.toml` or `setup.cfg`

**New Component/Module:**
- Implementation: Create `src/` directory for package structure
- Types: Inline or separate `types.py`
- Tests: `tests/test_<module>.py`

**New Command/CLI:**
- Definition: `main.py` or new `cli.py`
- Handler: Add argument parsing with click/argparse
- Tests: `tests/test_cli.py`

**Utilities:**
- Shared helpers: Extract to `utils.py` or `helpers.py`
- Type definitions: `types.py` or inline
